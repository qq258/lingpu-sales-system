# 手机销售后台管理系统 — 需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档根据项目实际实现，明确手机销售后台管理系统的功能需求、非功能需求及约束条件，为项目维护与后续迭代提供依据。本文档面向项目团队及干系人。

### 1.2 项目背景
某连锁手机销售门店拥有多个分店，日常管理手机产品的进货、库存、销售及门店间调货。各分店通过 Tailscale 组建虚拟局域网访问统一后台系统。系统支持多门店数据隔离，各门店仅操作本店数据，老板可查看全部门店经营数据，支持门店间调货。

### 1.3 系统组成
系统由三个子系统组成：

| 子系统 | 说明 | 端口 |
|--------|------|------|
| **管理后台 (client)** | 面向管理员/店长的完整后台管理 | 5173 |
| **门户网站 (portal)** | 面向店员的轻量操作门户 | 5174 |
| **后端服务 (server)** | 统一 API 服务 | 3000 |

### 1.4 术语与定义

| 术语 | 说明 |
|------|------|
| 型号 (model) | 手机的具体配置变体，含颜色、内存等信息。系统以型号为SKU最小管理单元 |
| 厂家条码 | 手机出厂时印在包装盒上的条形码 |
| IMEI | 国际移动设备识别码，每台手机唯一 |
| IMEI2 | 双卡手机的第二个 IMEI 号码 |
| SN 码 | 手机序列号 |
| CLodop | 云打印中间件，用于 Web 端直接打印小票 |
| Tailscale | 基于 WireGuard 的虚拟组网工具 |
| 调货 | 将一个门店的库存转移到另一门店 |

## 2. 总体需求

### 2.1 业务目标
- 统一管理所有分店业务
- 各门店数据隔离，店员仅操作本店数据
- 管理员可查看全部门店数据
- 支持门店间调货，调货后双方库存自动更新
- 扫码枪贯穿入库、销售等环节
- 支持 CLodop 小票打印
- 提供数据看板与销售统计

### 2.2 用户角色

| 角色 | 描述 | 主要权限范围 |
|------|------|-------------|
| **超级管理员** (super_admin) | 老板/系统所有者 | 全部功能；用户管理、门店管理、查看全部门店数据、调货审核 |
| **店长** (store_admin) | 各分店负责人 | 本店全部功能；本店用户管理、调货申请/接收 |
| **店员** (operator) | 日常业务操作员 | 本店产品查询、进货、销售、库存查询、打印小票、统计查看 |

> 每个用户绑定所属门店（超级管理员不绑定，可查看所有门店）。

## 3. 功能需求

### 3.1 基础鉴权系统

#### 3.1.1 用户登录
- 支持用户名/密码登录
- 密码 bcrypt 加密存储
- JWT Token 鉴权，有效期 24 小时
- 登录后自动识别所属门店，界面仅展示该门店数据
- 超级管理员登录后可获取所有门店列表，用于门店切换

#### 3.1.2 用户管理
- 管理员可创建、编辑、查询、删除用户
- 支持启用/禁用账户
- 支持重置用户密码
- 用户绑定所属门店
- 权限可见性规则：店员仅看店员，店长可看店员和店长，超级管理员看所有

### 3.2 门店管理
- 门店的新增、编辑、删除、查询（仅超级管理员）
- 字段：门店名称、编码、地址、联系电话、状态
- 删除门店前需确保门店下无用户

### 3.3 产品数据管理

#### 3.3.1 品牌管理
- 品牌的新增、编辑、删除、查询
- 品牌字段：名称、描述
- 删除品牌前需确保品牌下无型号

#### 3.3.2 手机型号管理（核心）
- 型号的新增、编辑、删除、查询
- 型号关联品牌
- 字段：型号名称、品牌、颜色、RAM、ROM、销售价格、进货成本价、厂家条码（唯一）、图片路径、上市年份、操作系统、网络制式、屏幕尺寸、CPU、电池容量、描述
- 支持厂家条码扫码查询
- 支持型号关键词搜索
- 型号为基础数据，全门店共享

### 3.4 进货管理

#### 3.4.1 供应商管理
- 供应商的新增、编辑、删除、查询
- 字段：供应商名称、联系人、联系电话、地址、备注

#### 3.4.2 采购入库
- **新建入库单**：选择供应商，生成入库单号
- **逐台添加 IMEI**：每调用一次添加一台手机的型号 + IMEI 记录，IMEI 系统全局唯一校验
- **批量粘贴 IMEI**：同一型号下批量粘贴多个 IMEI，自动过滤已存在的 IMEI
- **从入库单中移除手机**：未确认前可删除
- **快捷确认入库**：创建入库单的同时直接确认入库，一步完成
- **确认入库**：逐条校验 IMEI 唯一性，写入 IMEI 库存表，按型号汇总更新库存，写入库存流水

### 3.5 库存管理

#### 3.5.1 期初库存录入
- 选择型号、填写数量、确认后写入库存并标记来源为 initial
- 支持批量期初录入
- 写入库存流水

#### 3.5.2 库存查询
- 各门店查看本店实时库存（按型号汇总）
- 支持按关键词（条码/型号名称/颜色）搜索
- 支持按品牌、型号筛选
- 显示库存量、品牌、型号、颜色、存储、成本价、销售价

#### 3.5.3 IMEI 库存查询
- 查看每台手机的 IMEI 记录
- 支持按 IMEI、型号名称、条码搜索
- 显示 IMEI、IMEI2、SN码、状态、所属门店、入库时间
- 扫码 IMEI 快速查询

#### 3.5.4 库存流水
- 查看库存变动明细
- 支持按型号、变动类型筛选
- 显示变动类型（期初/入库/销售出库/调货出库/调货入库/盘点调整）、变动前后数量、操作人

#### 3.5.5 低库存预警
- 默认阈值 10 台，按库存量升序排列

#### 3.5.6 库存盘点
- 创建盘点单：录入各型号账面数量和实际盘点数量，自动计算差异
- 盘点单审核：根据差异自动调整库存，写入库存流水
- 仅可删除待审核的盘点单

### 3.6 门店间调货

#### 3.6.1 调货申请
- 选择目标门店、商品型号、数量、备注
- 系统自动校验来源门店库存是否充足
- 每型号生成一条调货记录

#### 3.6.2 调货出库（来源门店操作）
- 确认调出后扣减来源门店库存
- 写入库存流水（change_type = transfer_out）
- 调货单状态流转：pending → outbound

#### 3.6.3 调货入库（目标门店操作）
- 确认收货后增加目标门店库存
- 写入库存流水（change_type = transfer_in）
- 调货单状态流转：outbound → completed

#### 3.6.4 取消调货
- 待调出状态可直接取消
- 已调出状态取消时自动回滚库存
- 已完成状态不可取消

#### 3.6.5 调货单查询
- 按状态筛选
- 按门店过滤（来源店或目标店）

### 3.7 销售开单

#### 3.7.1 销售开单
- **商品选择**：扫码 IMEI 或手动输入 IMEI
- **IMEI 实时校验**：校验 IMEI 是否在当前门店且状态为 in_stock
- **多 IMEI 同单**：支持一个销售单包含多个不同型号的 IMEI
- **价格**：自动带入型号预设价格，可手动修改
- **实收金额**：可手动输入
- **客户信息**：可选输入客户姓名、地址、电话
- **确认保存**：
  - 自动生成订单编号（含门店标识）
  - 逐条标记 IMEI 状态为 sold
  - 按型号扣减库存
  - 写入库存流水（change_type = sale_out）

#### 3.7.2 打印小票
- 销售保存后可获取打印数据
- 支持补打
- 小票内容含门店名称、商品明细、IMEI、金额等

#### 3.7.3 销售记录
- 按时间倒序展示
- 支持按日期范围筛选
- 支持查看详情（含 IMEI 明细）
- 软删除（标记 status = 'deleted'）

### 3.8 数据统计与看板

#### 3.8.1 数据看板（管理后台）
- 今日销售额、今日单数、低库存数量
- 近 7 日销售趋势
- 热销型号排行（Top 10）
- 最近订单列表
- 今日门店销售对比（超级管理员）

#### 3.8.2 销售统计
- 按日/周/月维度统计销售额、销售单数、销售数量
- 支持按日期范围筛选
- 热销排行（支持时间范围）

#### 3.8.3 调货统计
- 调货总数
- 按状态分布
- 最近调货记录

### 3.9 门户网站（面向店员）

门户网站为独立的轻量客户端，面向非技术背景的一线店员，功能涵盖：

#### 3.9.1 数据看板（首页）
- 今日销售额、今日单数、低库存预警数
- 快捷操作入口（入库、查库存、开单）
- 销售额趋势图、售出排行图（支持时间范围切换）
- 今日最新售出记录

#### 3.9.2 采购入库（平铺式表单）
- 选择供应商
- 选择品牌→型号联动，填写单价
- 录入 IMEI（扫码枪或手动）
- 实时显示入库清单
- 确认入库

#### 3.9.3 库存查询
- 按关键词搜索 IMEI 库存
- 显示 IMEI 状态、型号信息、所属门店

#### 3.9.4 销售开单
- 扫码 IMEI 或手动搜索
- 自动填充商品信息
- 填写实收金额、客户信息
- 确认开单

#### 3.9.5 销售记录
- 按日期筛选查看历史记录
- 查看详情

### 3.10 文件上传
- 支持单文件和多文件上传（型号图片）
- 支持格式：jpg/jpeg/png/gif/webp
- 单文件大小限制：5MB

## 4. 非功能需求

### 4.1 性能需求
- 页面响应时间：常规操作 ≤ 2 秒
- 局域网环境，2-3 个并发用户

### 4.2 安全性
- 密码 bcrypt 加密存储
- 数据隔离：各门店只能查看和操作本店数据
- JWT Token 鉴权
- 操作权限按角色分级控制

### 4.3 可用性
- 管理后台侧边栏导航
- 门户网站顶部导航，扁平化路由
- 门户网站大字号、大按钮设计
- 关键操作二次确认

### 4.4 可维护性
- SQLite 数据库，单文件易备份
- Prisma ORM，数据库迁移管理
- pnpm workspaces 多包管理

### 4.5 兼容性
- 支持 Chrome、Edge 最新版本
- 门户网站适配大屏触摸操作

---

## 5. 约束条件

- **部署方式**：服务器电脑部署后端 + 数据库，各电脑安装 Tailscale 组网访问
- **数据库**：SQLite，存储在服务器电脑上
- **打印**：CLodop 插件安装在每台需要打印的电脑上
- **网络**：各分店通过 Tailscale 组成虚拟局域网
- **技术栈**：Express + Prisma + SQLite 后端，Vue 3 + Element Plus 前端
