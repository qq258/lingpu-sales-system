# 手机销售后台管理系统 — 概要设计文档

## 1. 项目概述

### 1.1 项目目标
构建一套统一的轻量化手机销售管理系统，部署在局域网内，支持多个分店同时使用。各门店数据隔离，支持门店间调货，管理员可查看全部门店经营数据。

### 1.2 系统组成

| 子系统 | 技术栈 | 端口 | 说明 |
|--------|--------|------|------|
| **server** | Express + Prisma + SQLite | 3000 | 统一后端 API 服务 |
| **client** | Vue 3 + Element Plus + TypeScript | 5173 | 管理后台（完整功能） |
| **portal** | Vue 3 + Element Plus + TypeScript | 5174 | 门户网站（面向店员，大字版） |

### 1.3 设计原则
- **统一部署**：一套后端覆盖所有分店
- **门店隔离**：通过 store_id 实现数据隔离
- **IMEI 级追踪**：每台手机通过 IMEI 全生命周期追踪
- **型号即 SKU**：以型号（含颜色、RAM、ROM）为 SKU 管理单元


## 4. 数据库设计

### 4.1 ER 关系

```
Store（门店） ─── 1:N ──── User（用户归属门店）
Store ─── 1:N ──── Inventory（库存按门店独立）
Store ─── 1:N ──── InventoryImei（IMEI按门店独立）
Store ─── 1:N ──── SaleOrder（销售单按门店独立）
Store ─── 1:N ──── PurchaseEntry（入库单按门店独立）
Store ─── 1:N ──── Transfer（调货单，来源+目标门店）

Brand（品牌） ─── 1:N ──── Model（型号，含颜色/RAM/ROM/价格）
Model ──── Inventory（库存汇总，model_id + store_id 唯一）
Model ──── InventoryImei（IMEI库存明细）
```

### 4.2 核心数据表清单

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| `sys_store` | 门店表 | id, name, code, address, phone, status |
| `sys_user` | 用户表 | id, username, password(bcrypt), real_name, role, store_id, status |
| `sys_operation_log` | 操作日志表 | id, user_id, store_id, action, module, target_id, detail, ip |
| `pdt_brand` | 品牌表 | id, name, description, status |
| `pdt_model` | 型号表（SKU） | id, brand_id, name, color, ram, rom, sale_price, cost_price, manufacturer_barcode(unique), image_path, 规格字段... |
| `pch_supplier` | 供应商表 | id, name, contact_person, phone, address, remark |
| `pch_purchase_entry` | 入库单表 | id, entry_no(unique), store_id, supplier_id, total_amount, status(pending/confirmed), operator_id |
| `pch_purchase_item` | 入库单项表（IMEI级） | id, entry_id, model_id, imei, imei2, sn_code, unit_price |
| `wh_inventory` | 库存汇总表 | id, model_id, store_id(unique复合), quantity, brand_name, model_name, color, storage, cost_price, sale_price |
| `wh_inventory_log` | 库存流水表 | id, model_id, store_id, change_type, qty_before/qty_change/qty_after, ref_type, ref_id, operator_id |
| `wh_inventory_imei` | IMEI库存表（核心） | id, model_id, store_id, imei(unique), imei2, sn_code, status(in_stock/sold), entry_id, sold_at |
| `wh_inventory_check` | 盘点单表 | id, check_no(unique), store_id, status(pending/completed), checker_id |
| `wh_inventory_check_item` | 盘点明细表 | id, check_id, model_id, expected_qty, actual_qty, diff_qty |
| `sale_order` | 销售单表 | id, order_no(unique), store_id, model_id, model_name, quantity, unit_price, total_amount, actual_amount, operator_id, status(active/deleted) |
| `sale_order_item` | 销售单项表（IMEI级） | id, sale_order_id, model_id, model_name, imei, imei2, sn_code, quantity, unit_price |
| `wh_transfer` | 调货单表 | id, transfer_no(unique), from_store_id, to_store_id, model_id, quantity, status(pending/outbound/completed/cancelled) |

### 4.3 核心设计要点

> **IMEI 为核心资产**：每台手机从入库到销售，全生命周期通过 `wh_inventory_imei` 追踪

> **库存数量计算**：`wh_inventory.quantity` 由入库/销售/调货/盘点操作时同步更新，而非实时 COUNT

> **型号即 SKU**：型号（Model）直接作为销售单元，包含颜色、RAM、ROM、价格等属性，无独立的 SKU 表

## 5. 关键业务流程

### 5.1 门店数据隔离机制

```
用户登录 → JWT payload 包含 userId, role, storeId
  → store-scope 中间件处理：
    → 超级管理员：可从 query/body 取 store_id 覆盖，查看任意门店
    → 普通用户：强制使用 JWT 中的 storeId
  → 所有 API 查询自动带上 store_id 条件
  → 新增数据自动填充 store_id
```

### 5.2 采购入库流程

```
① 新建入库单 → 选择供应商 → 生成入库单号 (PO + 门店编码 + 日期 + 流水)
② 逐台添加 IMEI：
   → 提交 model_id + imei + unit_price
   → IMEI 唯一性校验（全局不可重复）
   → 写入 pch_purchase_item
③ 可批量粘贴 IMEI（同型号多个 IMEI）
④ 确认入库（事务）：
   → 逐条校验 IMEI 唯一性
   → 逐条写入 wh_inventory_imei (status='in_stock')
   → 按 model_id 汇总更新 wh_inventory.quantity
   → 写入 wh_inventory_log (change_type='purchase_in')
   → 更新入库单 status='confirmed'
⑤ 快捷入库：创建 + 确认一步完成
```

### 5.3 调货流程

```
分店 A（库存不足）          分店 B（有库存）
    │                            │
    ├── 发起调货申请              │
    │   选择目标门店=B            │
    │   选择型号+数量             │
    │   状态: pending              │
    │                            │
    │                            ├── 确认调出
    │                            │   扣减 B 店库存
    │                            │   change_type='transfer_out'
    │                            │   状态: outbound
    │                            │
    ├── 确认收货                  │
    │   增加 A 店库存              │
    │   change_type='transfer_in' │
    │   状态: completed            │
```

### 5.4 销售开单流程

```
① 扫码或输入 IMEI → 校验 IMEI 在当前门店且 status='in_stock'
② 添加多个 IMEI 到购物篮（可跨型号）
③ 设置实收金额、客户信息
④ 确认开单（事务）：
   → 生成订单号 (SA + 门店编码 + 日期 + 流水)
   → 创建 sale_order + sale_order_item（IMEI 明细）
   → 逐条标记 wh_inventory_imei.status='sold'
   → 按 model_id 扣减 wh_inventory.quantity
   → 写入 wh_inventory_log (change_type='sale_out')
⑤ 获取打印数据
```

- 响应格式：`{ code, message, data }`
