# 手机销售后台管理系统 — 详细设计说明书

## 1. 引言

### 1.1 编写目的
本文档在概要设计的基础上，对手机销售后台管理系统的各个模块进行详细设计，明确数据库字段规格、API 请求/响应格式、前端组件结构、业务逻辑实现细节、错误处理机制等，为开发人员提供可直接编码的详细指导。本文档基于实际代码实现编写，反映当前系统的真实状态。

### 1.2 适用范围
本文档面向项目开发团队，作为编码实现和后续维护的技术依据。

### 1.3 术语与约定

| 术语 | 说明 |
|------|------|
| model_id | 型号ID，替代原有的 sku_id，型号即为 SKU |
| change_type | 库存变动类型：purchase_in（入库）、initial（期初）、sale_out（销售出库）、transfer_out（调货出库）、transfer_in（调货入库）、transfer_cancel（调货取消回滚）、check（盘点调整） |
| 单号前缀 | PO=入库单, SA=销售单, CK=盘点单, TF=调货单 |

## 2. 数据库详细设计

### 2.1 数据库选型
- **数据库**：SQLite 3.x
- **ORM**：Prisma (Prisma Client JS)
- **命名规范**：表名使用 `snake_case` 加模块前缀，字段使用 `snake_case`

## 3. API 详细设计

### 3.1 通用规范

- **基础路径**：`/api/v1`
- **请求头**：`Authorization: Bearer <token>`（除登录接口外都需要）
- **当前门店**：
  - 普通用户：从 JWT 中解析 `store_id`，自动应用到数据查询
  - 超级管理员：通过请求参数 `store_id` 或 `storeId` 切换门店
- **统一响应格式**：

```json
// 成功
{ "code": 200, "message": "success", "data": {} }

// 分页
{ "code": 200, "message": "success", "data": { "list": [], "total": 0, "page": 1, "pageSize": 20 } }

// 客户端错误
{ "code": 400, "message": "参数错误: xxx", "data": null }

// 未授权
{ "code": 401, "message": "未登录或Token已过期", "data": null }

// 无权限
{ "code": 403, "message": "无权限操作", "data": null }

// 服务端错误
{ "code": 500, "message": "服务器内部错误", "data": null }
```

## 4. 单号生成规则

| 单据类型 | 前缀 | 规则 | 示例 |
|---------|------|------|------|
| 入库单 | PO | PO + 门店编码 + yyyyMMdd + 4位流水 | POA202606140001 |
| 销售单 | SA | SA + 门店编码 + yyyyMMdd + 4位流水 | SAA202606140001 |
| 盘点单 | CK | CK + 门店编码 + yyyyMMdd + 4位流水 | CKA202606140001 |
| 调货单 | TF | TF + 门店编码 + yyyyMMdd + 4位流水 - 序号 | TFA202606140001-1 |

> **实现逻辑**：`server/src/utils/order-no.ts` — 根据前缀查询当日最大流水号 +1。

## 5. 业务事务逻辑

### 5.1 确认入库事务

```
BEGIN TRANSACTION
  FOR EACH item IN pch_purchase_item:
    ① 校验 imei 在 wh_inventory_imei 中唯一
    ② IF 重复 THEN ROLLBACK（整单回滚）
    ③ INSERT INTO wh_inventory_imei (model_id, store_id, imei, imei2, sn_code, status='in_stock', entry_id)
  END FOR
  ④ 更新入库单 status='confirmed', total_amount
  ⑤ 按 model_id 分组：
     FOR EACH group:
       UPDATE wh_inventory SET quantity = quantity + count
       INSERT INTO wh_inventory_log (change_type='purchase_in')
COMMIT
```

### 5.2 调货出库事务

```
BEGIN TRANSACTION
  ① 校验库存充足
  ② UPDATE wh_inventory SET quantity = quantity - transfer.quantity
  ③ INSERT INTO wh_inventory_log (change_type='transfer_out', qty_change=-N)
  ④ UPDATE wh_transfer SET status='outbound', outbound_operator_id, outbound_at
COMMIT
```

### 5.3 调货入库事务

```
BEGIN TRANSACTION
  ① IF wh_inventory 存在 THEN UPDATE quantity = quantity + N ELSE INSERT
  ② INSERT INTO wh_inventory_log (change_type='transfer_in', qty_change=+N)
  ③ UPDATE wh_transfer SET status='completed', inbound_operator_id, inbound_at
COMMIT
```

### 5.4 销售开单事务

```
BEGIN TRANSACTION
  ① FOR EACH imei:
      校验 wh_inventory_imei.status='in_stock' AND store_id=当前门店
      UPDATE wh_inventory_imei SET status='sold', sold_at=now()
  ② CREATE sale_order + sale_order_items
  ③ 按 model_id 分组扣减 wh_inventory
  ④ INSERT INTO wh_inventory_log (change_type='sale_out')
COMMIT
```
