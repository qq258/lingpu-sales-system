# 手机销售后台管理系统 — 详细设计说明书

## 1. 引言

### 1.1 编写目的
本文档在概要设计的基础上，对手机销售后台管理系统的各个模块进行详细设计，明确数据库字段规格、API 请求/响应格式、前端组件结构、业务逻辑实现细节、错误处理机制等，为开发人员提供可直接编码的详细指导。

### 1.2 适用范围
本文档面向项目开发团队，作为编码实现的技术依据。

### 1.3 术语与约定

| 术语 | 说明 |
|------|------|
| CRUD | 增删改查操作（Create / Read / Update / Delete） |
| JWT | JSON Web Token，用户认证令牌 |
| store_id | 门店ID，所有业务数据通过该字段隔离 |
| change_type | 库存变动类型：in（入库）、out（出库）、initial（期初）、adjust（盘点调整）、transfer_in（调货入库）、transfer_out（调货出库） |

---

## 2. 数据库详细设计

### 2.1 数据库选型
- **数据库**：SQLite 3.x
- **ORM**：TypeORM 或 Prisma
- **命名规范**：表名使用 `snake_case` 加模块前缀，字段使用 `snake_case`

## 3. API 详细设计

### 3.1 通用规范

- **基础路径**：`/api/v1`
- **请求头**：`Authorization: Bearer <token>`（除登录接口外都需要）
- **当前门店**：从 JWT 中解析 `store_id`，自动应用到数据查询（超级管理员可通过请求参数覆盖）
- **统一响应格式**：

```json
// 成功
{ "code": 200, "message": "success", "data": {} }

// 分页
{ "code": 200, "message": "success", "data": { "list": [], "total": 0, "page": 1, "pageSize": 10 } }

// 客户端错误
{ "code": 400, "message": "参数错误: xxx", "data": null }

// 未授权
{ "code": 401, "message": "未登录或Token已过期", "data": null }

// 无权限
{ "code": 403, "message": "无权限操作", "data": null }

// 服务端错误
{ "code": 500, "message": "服务器内部错误", "data": null }
```

### 3.4 产品管理

#### CRUD /brands, /models, /skus
标准 CRUD，全门店共享数据。

#### GET /skus/scan?barcode={条码}
扫码枪查询接口，**核心高频接口**。

```json
// Request
GET /api/v1/skus/scan?barcode=6901234567890

// Response — 匹配成功
{
  "code": 200,
  "data": {
    "id": 1,
    "sku_code": "SKU202506010001",
    "model_name": "iPhone 17 Pro Max",
    "brand_name": "Apple",
    "color": "深黑色",
    "ram": "8GB",
    "rom": "256GB",
    "sale_price": 6999.00,
    "cost_price": 5800.00,
    "stock_quantity": 15,
    "barcode": "6901234567890",
    "image_path": "/uploads/iphone17_black.jpg"
  }
}

// Response — 条码未识别
{ "code": 404, "message": "未识别商品", "data": null }
```

**查询逻辑**：
```
① 查询 pdt_sku_barcode WHERE barcode = ? → 获取 sku_id
② 如果没找到，查询 pdt_sku WHERE manufacturer_barcode = ? → 获取 sku_id
③ 如果还没找到，返回 404
④ 根据 sku_id 查询当前用户所属门店的库存量
⑤ 返回 SKU 详情 + 库存量
```

### 3.5 进货管理

#### POST /purchase-entries/:id/scan
扫码入库接口，每次扫描一台上送一次。

```json
// Request
{ "barcode": "6901234567890" }

// Response — 匹配成功，返回当前累计
{
  "code": 200,
  "data": {
    "entry_id": 1,
    "items": [
      { "sku_id": 1, "sku_name": "iPhone 17 Pro Max 256GB 深黑", "quantity": 2 },
      { "sku_id": 2, "sku_name": "iPhone 17 Pro Max 512GB 白色", "quantity": 1 }
    ],
    "total_items": 3,
    "last_scan": {
      "barcode": "6901234567890",
      "matched": true,
      "sku_name": "iPhone 17 Pro Max 256GB 深黑"
    }
  }
}

// Response — 条码未识别
{ "code": 404, "message": "未识别商品", "data": null }
```

#### PUT /purchase-entries/:id/confirm
确认入库。

```json
// Request — 为每个 SKU 填写进货单价
{
  "items": [
    { "sku_id": 1, "unit_price": 5800 },
    { "sku_id": 2, "unit_price": 6200 }
  ]
}

// Response
{
  "code": 200,
  "data": {
    "entry_id": 1,
    "entry_no": "RKA202506100001",
    "total_amount": 17800.00,
    "items_count": 2,
    "created_at": "2025-06-10 14:30:00"
  }
}
```

**后端事务逻辑**：
```
BEGIN TRANSACTION
  FOR EACH item IN items:
    ① 查询 wh_inventory WHERE sku_id=? AND store_id=当前门店
    ② IF 存在 THEN UPDATE quantity = quantity + item.quantity
       ELSE INSERT (sku_id, store_id, quantity) VALUES (?, 当前门店, item.quantity)
    ③ 写入 wh_inventory_log (change_type='in', ref_type='purchase')
  END FOR
  ④ 更新入库单状态为 done
COMMIT
```

### 5.1 订单编号生成规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 销售单号 | `XS` + 门店编码 + yyyyMMdd + 4位流水 | `XSA202506100001` |
| 入库单号 | `RK` + 门店编码 + yyyyMMdd + 4位流水 | `RKA202506100001` |
| 调货单号 | `TH` + yyyyMMdd + 4位流水 | `TH202506100001` |
| 盘点单号 | `PD` + 门店编码 + yyyyMMdd + 4位流水 | `PDA202506100001` |

**实现**：
```typescript
async function generateOrderNo(prefix: string, storeCode: string): Promise<string> {
  const date = new Date()
  const dateStr = format(date, 'yyyyMMdd')
  const todayPrefix = `${prefix}${storeCode}${dateStr}`
  
  // 查询当天已使用的最大流水号
  const lastOrder = await this.repository.findOne({
    where: { order_no: Like(`${todayPrefix}%`) },
    order: { order_no: 'DESC' }
  })
  
  const seq = lastOrder 
    ? parseInt(lastOrder.order_no.slice(-4)) + 1 
    : 1
    
  return `${todayPrefix}${String(seq).padStart(4, '0')}`
}
```

### 5.2 扫码枪输入识别（scanner.ts）

```typescript
// scanner.ts  — 检测扫码枪输入还是手动键盘输入
// 扫码枪特性：快速连续输入（间隔<30ms）+ 以回车结尾

export function createScanner(options: {
  onScan: (barcode: string) => void
  timeout?: number  // 默认 100ms
}) {
  let buffer = ''
  let lastTime = 0
  const timeout = options.timeout || 100

  function handleKeydown(e: KeyboardEvent) {
    const now = Date.now()
    
    if (e.key === 'Enter') {
      if (buffer.length >= 8) { // 扫码枪输入至少8位字符
        options.onScan(buffer)
      }
      buffer = ''
      lastTime = 0
      return
    }
    
    // 判断是扫码枪（间隔 < 30ms）还是手动输入
    if (now - lastTime > 0 && now - lastTime < 30) {
      buffer += e.key
    } else {
      buffer = e.key // 手动输入，重置
    }
    lastTime = now
    
    // 超时未输入完成视为手动输入
    setTimeout(() => {
      if (now === lastTime && buffer.length > 0 && buffer.length < 8) {
        // 手动输入的短内容，不做扫码处理
        buffer = ''
      }
    }, timeout)
  }

  return {
    attach: (el: HTMLElement) => el.addEventListener('keydown', handleKeydown),
    detach: (el: HTMLElement) => el.removeEventListener('keydown', handleKeydown)
  }
}
```

### 5.3 CLodop 打印封装（clodop.ts）

```typescript
// clodop.ts — 小票打印工具

interface PrintData {
  storeName: string
  orderNo: string
  createdAt: string
  operatorName: string
  items: Array<{
    skuName: string
    quantity: number
    unitPrice: number
  }>
  totalAmount: number
  actualAmount: number
  customerName?: string
}

/**
 * 获取 CLodop 对象
 */
function getLodop(): any {
  const LODOP = (window as any).getLodop?.()
  if (!LODOP) {
    throw new Error('CLodop 未安装或未启动，请检查')
  }
  return LODOP
}

/**
 * 打印销售小票
 */
export function printReceipt(data: PrintData): void {
  const LODOP = getLodop()
  
  LODOP.PRINT_INIT('手机销售小票')
  LODOP.SET_PRINT_PAGESIZE(1, 100, 150, '') // 宽度100mm, 高度自适应
  
  let top = 10
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 20, `${data.storeName}`)
  LODOP.SET_PRINT_STYLEA(0, 'FontSize', 14)
  LODOP.SET_PRINT_STYLEA(0, 'Bold', 1)
  
  top += 25
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 15, '━━━ 销售小票 ━━━')
  
  top += 20
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `单号: ${data.orderNo}`)
  top += 15
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `时间: ${data.createdAt}`)
  top += 15
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `操作员: ${data.operatorName}`)
  
  top += 20
  data.items.forEach(item => {
    LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `${item.skuName} × ${item.quantity}`)
    top += 15
  })
  
  top += 10
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `━━━━━━━━━━━━━━`)
  top += 15
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `应收: ¥${data.totalAmount.toFixed(2)}`)
  top += 15
  
  if (data.totalAmount !== data.actualAmount) {
    LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `实收: ¥${data.actualAmount.toFixed(2)}`)
    LODOP.SET_PRINT_STYLEA(0, 'Bold', 1)
    top += 15
  }
  
  if (data.customerName) {
    LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, `客户: ${data.customerName}`)
    top += 15
  }
  
  top += 10
  LODOP.ADD_PRINT_TEXT(top, 10, 200, 12, '感谢您的惠顾！')
  
  LODOP.PRINT()
}
```

// Nest.js 中间件示例
@Injectable()
export class StoreScopeMiddleware implements NestMiddleware {
  use(req: any, res: any, next: () => void) {
    const user = req.user // 从 JWT 解析
    
    if (user.role === 'super_admin') {
      // 超级管理员可通过请求参数覆盖门店
      req.effectiveStoreId = req.query.store_id 
        ? parseInt(req.query.store_id) 
        : null // null = 全部门店
    } else {
      // 普通用户固定为本门店
      req.effectiveStoreId = user.store_id
      
      // 如果修改类 API 传了 store_id，校验是否等于 user.store_id
      if (req.method !== 'GET' && req.body.store_id 
          && req.body.store_id !== user.store_id) {
        throw new ForbiddenException('只能操作本店数据')
      }
    }
    
    next()
  }
}

## 7. 前端 FAQ / 注意事项

### 7.1 关于 CLodop 打印
- CLodop 需要**在每台需要打印的电脑上独立安装**
- 前端通过 `window.getLodop()` 检测是否安装
- 如果未安装，应给出友好的提示文案和下载链接
- CLodop 官网提供免费版本，搜索 "CLodop" 即可

### 7.2 关于扫码枪
- 市面上常见的 USB 扫码枪默认为 HID 键盘模式，即模拟键盘输入
- 本系统的 scanner.ts 通过检测字符输入间隔来判断是否为扫码枪
- 如果扫码枪型号特殊（如 USB 串口模式），请调整 scanner.ts 的参数
- 建议所有 SKU 录入后先扫码测试一遍，确保条码正确

### 7.3 关于 SQLite 并发
- SQLite 在低并发场景下表现良好
- 本系统最多 3 个用户并发，SQLite 足够胜任
- 如未来门店扩张到 5 个以上，建议切换为 PostgreSQL

### 7.4 期初库存录入注意事项
- 期初库存录入仅在系统启用时使用一次
- 录入后如需调整，应通过"库存盘点"功能修改
- 建议录入前先对门店实物进行一次全面盘点
- 批量录入时建议先导出 SKU 列表，线下盘点后填写导入
