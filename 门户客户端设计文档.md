# 手机销售门户网站 — 客户端设计文档

## 1. 概述

### 1.1 背景
现有系统包含一个 **Express + Prisma 后端**（端口 3000）和一个 **Vue 3 + Element Plus 后台管理客户端**（端口 5173）。管理客户端功能全面但面向管理员，操作路径偏长、字体偏小，不适合非技术背景的一线店员使用。

### 1.2 目标
提供**独立的门户网站客户端**，面向**门店店员、仓管员、收银员等非技术人员**，要求：

- **核心功能**：看板（数据一目了然）、入库（采购入库）、库存查询、销售开单、销售记录
- **易用性**：小学毕业也能丝滑操作，不费眼
- **视觉风格**：干净简洁，字大、按钮大、信息层级清晰
- **技术复用**：与现有后端共享 API，仅依赖一个轻量排行接口

### 1.3 与现有客户端的定位差异

| 维度 | 现有管理后台 (client) | 门户网站 (portal) |
|------|-------------------|------------------|
| 目标用户 | 管理员、老板 | 店员、仓管员、收银员 |
| 功能范围 | 全功能（品牌型号、供应商、门店、用户等管理） | 5 个核心页面：看板、入库、库存查询、销售开单、销售记录 |
| 操作复杂度 | 菜单层级深，操作路径长 | 扁平化，一步直达 |
| 字体大小 | 标准（12-14px） | 大号（18-24px 正文，32-48px 关键数字） |
| 按钮/交互 | 标准尺寸 | 超大按钮，触摸友好 |
| 导航 | 侧边栏菜单 | 顶部固定导航栏 |
| 入口 | 管理后台完整入口 | 门店快捷操作入口 |

## 2. 技术方案

### 2.1 整体架构

```
┌─────────────────────────────────────────────┐
│              门户客户端 (portal)              │
│  Vue 3 + Vite + Element Plus 大字主题         │
│  单页面应用，运行在 5174 端口                  │
├─────────────────────────────────────────────┤
│               共享后端服务 (server)            │
│  Express + Prisma + SQLite (端口 3000)       │
│  API 基础路径: /api/v1                        │
└─────────────────────────────────────────────┘
```

### 2.2 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | Vue 3 (Composition API + `<script setup>`) | 与现有客户端一致 |
| 构建 | Vite 5 | 与现有客户端一致 |
| 语言 | TypeScript | 与现有客户端一致 |
| UI 组件库 | Element Plus（大字版 CSS 定制） | 组件大号化定制 |
| 路由 | Vue Router 4 | 6 个页面，扁平化 |
| 状态管理 | Pinia | 用户状态 + 当前门店 |
| HTTP | Axios | 统一 token 注入 + 错误处理 |
| 样式 | 独立全局 CSS + glassmorphism 设计 | 磨砂玻璃效果，大字号 |

### 2.4 依赖的 API 清单

门户客户端仅使用后端已有接口，所有读写操作均通过后端统一鉴权和数据隔离机制。

| 功能 | 请求方式 | API 端点 | 说明 |
|------|---------|---------|------|
| 登录 | POST | `/auth/login` | 返回 token + 用户信息 |
| 获取用户信息 | GET | `/auth/userinfo` | 刷新当前用户信息 |
| 看板数据 | GET | `/stats/dashboard` | 今日销售/单数/低库存/趋势/排行 |
| 销售趋势 | GET | `/stats/sales` | 支持 timeRange 聚合 |
| 热销排行 | GET | `/stats/top-products` | 支持时间范围筛选 |
| 供应商列表 | GET | `/purchase/suppliers` | 采购入库选择供应商 |
| 快捷确认入库 | POST | `/purchase/purchase-entries/quick-confirm` | 一步创建+确认入库 |
| 品牌列表 | GET | `/products/brands` | 型号选择联动 |
| 型号列表 | GET | `/products/models` | 按品牌筛选 |
| IMEI 库存列表 | GET | `/inventory/imei-list` | 库存查询页 |
| 扫码查询 IMEI | GET | `/inventory/scan-imei` | 销售开单校验 IMEI |
| 创建销售单 | POST | `/sales/sales` | 销售开单 |
| 销售列表 | GET | `/sales/sales` | 销售记录页 |
| 打印数据 | GET | `/sales/sales/:id/print-data` | 小票打印预览 |

## 3. 页面结构

### 3.1 路由设计

```javascript
const routes = [
  { path: '/login', component: Login },
  {
    path: '/',
    component: Layout,           // TopNav 布局
    children: [
      { path: '', redirect: '/dashboard' },
      { path: 'dashboard',     component: Dashboard },     // 首页数据看板
      { path: 'entry',         component: Entry },         // 采购入库
      { path: 'inventory',     component: Inventory },     // 库存查询
      { path: 'sale',          component: Sale },          // 销售开单
      { path: 'sales-record',  component: SalesRecord },   // 销售记录
    ],
  },
]
```

共 6 个页面，所有需要登录的页面使用 Layout 包裹（含 TopNav）。

### 3.2 路由守卫

```javascript
router.beforeEach(async (to, _from, next) => {
  const userStore = useUserStore()
  if (to.meta.requiresAuth) {
    if (!userStore.token) { next('/login'); return }
    if (!userStore.userInfo) {
      try { await userStore.fetchUserInfo() } catch { next('/login'); return }
    }
  }
  next()
})
```

页面鉴权由路由守卫统一控制，首次进入时自动调用 `/auth/userinfo` 恢复会话。

**操作流程**：

```
① IMEI 输入框聚焦 → 扫码枪扫描 IMEI 条码 → 自动查询校验
② 校验通过 → 商品加入购物车列表（自动显示品牌/型号/规格）
③ 单价可手动修改（默认取型号 sale_price）
④ 重复①-③直到所有商品扫码完毕
⑤ 系统自动计算总金额，收款金额默认等于总金额
⑥ 填写客户信息（可选）
⑦ 点击「确认收款」→ 二次确认 → 调用 createSale API
```

- IMEI 的 status 必须为 `in_stock`
- IMEI 必须属于当前门店

## 5. 门店数据隔离机制

门户客户端不实现门店切换 UI（与管理后台不同），因为门户面向的操作员用户固定属于一个门店。但代码层仍通过 `userStore.effectiveStoreId` 支持：

```javascript
// 所有 API 调用时自动附加 storeId 参数
const params: any = {}
if (userStore.effectiveStoreId) params.storeId = userStore.effectiveStoreId
const result = await getInventoryImeiList(params)
```

对于超级管理员使用门户场景，`effectiveStoreId` 在登录时自动设置为 stores 列表中的第一个门店ID。

## 6. 状态管理

### 6.1 userStore

```typescript
// stores/user.ts
state:
  token           — JWT Token（从 localStorage 持久化）
  userInfo        — 用户信息 { id, username, realName, role, storeId }
  currentStoreId  — 当前操作门店ID（普通用户=固定，超级管理员=首个门店）
  stores          — 门店列表（仅超级管理员有值）

getters:
  isLoggedIn          — !!token
  effectiveStoreId    — currentStoreId（所有 API 调用使用此值）

actions:
  login(username, password)     — POST /auth/login，解析 userInfo 和 stores
  fetchUserInfo()               — GET /auth/userinfo，刷新用户会话
  logout()                      — 清除所有状态和 localStorage
  setStore(storeId)             — 切换当前门店（预留）
```

### 7.2 设计语言

- **磨砂玻璃效果**（glassmorphism）：所有卡片使用 `backdrop-filter: blur(16px)` + 半透明背景
- **文字层级**：关键数字 36px Bold，标题 26px Bold，正文 16-17px，辅助信息 13-14px
- **按钮**：业务操作按钮高度 52-60px，圆角 14px，大字号 17-20px
- **间距**：页面边距 32px，卡片内边距 22-24px，卡片间距 20px

### 7.3 响应式

门户网站主要适配桌面端（1920×1080 及以上），页面内容区域最大宽度 1400px，居中显示。
