# 手机销售后台管理系统 — API 接口文档

> 本文档基于实际代码实现自动生成，反映系统当前真实接口状态。
>
> **基础路径**：`/api/v1`
>
> **认证方式**：`Authorization: Bearer <token>`（除登录接口外都需要）
>
> **统一响应格式**：
> ```json
> { "code": 200, "message": "success", "data": {} }
> { "code": 400, "message": "参数错误", "data": null }
> { "code": 401, "message": "未登录或Token已过期", "data": null }
> { "code": 403, "message": "无权限操作", "data": null }
> { "code": 500, "message": "服务器内部错误", "data": null }
> ```
>
> **分页响应格式**：
> ```json
> { "code": 200, "message": "success", "data": { "list": [], "total": 0, "page": 1, "pageSize": 20 } }
> ```
>
> **门店隔离**：普通用户自动使用 JWT 中的 store_id 过滤数据；超级管理员可通过 query 参数 `store_id` 或 `storeId` 切换视角。

## 1. 鉴权模块 (Auth)

路由前缀：`/api/v1/auth`

### 1.1 用户登录

```
POST /api/v1/auth/login
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 登录用户名 |
| password | string | 是 | 登录密码 |

**响应示例**（超级管理员）：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "userInfo": {
      "id": 1,
      "username": "admin",
      "realName": "张三",
      "role": "super_admin",
      "storeId": null
    },
    "stores": [
      { "id": 1, "name": "分店A", "code": "A" },
      { "id": 2, "name": "分店B", "code": "B" }
    ]
  }
}
```

**响应示例**（普通用户，无 stores 字段）：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "userInfo": {
      "id": 2,
      "username": "user1",
      "realName": "李四",
      "role": "operator",
      "storeId": 1
    }
  }
}
```

### 1.2 获取用户信息

```
GET /api/v1/auth/userinfo
```

**请求头**：`Authorization: Bearer <token>`

**响应**：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "admin",
    "real_name": "张三",
    "role": "super_admin",
    "store_id": null,
    "status": 1,
    "created_at": "2025-01-01T00:00:00.000Z",
    "store": null,
    "stores": [
      { "id": 1, "name": "分店A", "code": "A" },
      { "id": 2, "name": "分店B", "code": "B" }
    ]
  }
}
```

### 1.3 获取用户列表

```
GET /api/v1/auth/users
```

**权限**：超级管理员（全部）、店长（本店）、店员（本店同级）

**响应**：

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "username": "admin",
        "realName": "张三",
        "role": "super_admin",
        "storeId": null,
        "storeName": null,
        "status": 1,
        "createdAt": "2025-01-01T00:00:00.000Z"

  }
}
```

### 1.4 创建用户

```
POST /api/v1/auth/register
```

**权限**：仅超级管理员

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 登录用户名 |
| password | string | 是 | 密码（至少6位） |
| realName | string | 是 | 真实姓名 |
| role | string | 否 | 角色，默认 operator |
| storeId | number | 否 | 所属门店ID |

**响应**：

```json
{ "code": 200, "data": { "id": 3, "username": "user2", "real_name": "王五", "role": "operator", "store_id": 1 } }
```

### 1.5 编辑用户

```
PUT /api/v1/auth/users/:id
```

**权限**：仅超级管理员

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 否 | 登录用户名 |
| realName | string | 否 | 真实姓名 |
| role | string | 否 | 角色 |
| storeId | number | 否 | 所属门店ID |

### 1.6 重置密码

```
PUT /api/v1/auth/users/:id/reset-password
```

**权限**：仅超级管理员

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| password | string | 是 | 新密码（至少6位） |

### 1.7 删除用户

```
DELETE /api/v1/auth/users/:id
```

**权限**：仅超级管理员（不能删除自己，不能删除超级管理员角色）

```
DELETE /api/v1/purchase/purchase-entries/:entryId/items/:itemId

```json

```

### 4.14 快捷确认入库

```
POST /api/v1/purchase/purchase-entries/quick-confirm
```

一步完成创建入库单 + 确认入库流程。

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplier_id | number | 否 | 供应商ID |
| remark | string | 否 | 备注 |
| items | object[] | 是 | 商品列表 |
| items[].model_id | number | 是 | 型号ID |
| items[].imei | string | 是 | IMEI |
| items[].imei2 | string | 否 | IMEI2 |
| items[].sn_code | string | 否 | SN码 |
| items[].unit_price | number | 否 | 单价 |

### 4.15 确认入库

```
PUT /api/v1/purchase/purchase-entries/:id/confirm
```

**事务逻辑**：
1. 逐条校验 IMEI 唯一性
2. 逐条写入 `wh_inventory_imei`（status='in_stock'）
3. 按型号汇总更新 `wh_inventory.quantity`
4. 写入 `wh_inventory_log`（change_type='purchase_in'）
5. 更新入库单 status='confirmed'

**错误**：
- `{ "code": 400, "message": "入库单已确认" }`
- `{ "code": 400, "message": "入库单无商品，无法确认" }`

```

```

**说明**：用于销售开单时扫码校验 IMEI。校验 IMEI 是否存在、是否 in_stock、是否属于当前门店。

**响应**：

```json
{
  "code": 200,
  "data": {
    "id": 1, "modelId": 1, "imei": "356938123456789",
    "imei2": null, "sn_code": null,
    "brandName": "Apple", "modelName": "iPhone 17 Pro Max",

```

- `{ "code": 404, "message": "未找到该IMEI对应的库存记录" }`
- `{ "code": 400, "message": "该IMEI(xxx)已售出，无法销售" }`
- `{ "code": 400, "message": "该IMEI(xxx)不属于当前门店" }`

### 5.3 IMEI 信息查询（完整追踪链路）

```
GET /api/v1/inventory/imei-query?imei=356938123456789
```

**说明**：按 IMEI 查询手机完整追踪链路信息，包括基础信息、入库记录、销售记录。与 scan-imei 不同，该接口不校验门店归属、不校验是否已售，返回该 IMEI 在系统中的全部关联数据。

**查询参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| imei | string | 是 | 完整的 IMEI 码 |

**响应示例（已售）**：

```json
{
  "code": 200,
  "data": {
    "id": 1, "modelId": 1,

```
GET /api/v1/transfers/:id
```

### 6.4 确认调出

```
PUT /api/v1/transfers/:id/outbound
```

**权限**：来源门店操作

**事务**：
1. 校验库存充足
2. 扣减来源门店 `wh_inventory.quantity`
3. 写入库存流水（change_type='transfer_out'）
4. 更新调货单 status='outbound'

### 6.5 确认收货

```
PUT /api/v1/transfers/:id/inbound
```

**权限**：目标门店操作

**事务**：
1. 增加/创建目标门店 `wh_inventory`
2. 写入库存流水（change_type='transfer_in'）
3. 更新调货单 status='completed'

### 6.6 取消调货

```
PUT /api/v1/transfers/:id/cancel
```

**说明**：
- 待调出（pending）：直接取消
- 已调出（outbound）：自动回滚库存后再取消
- 已完成（completed）：不可取消

## 7. 销售管理 (Sales)

路由前缀：`/api/v1/sales`

### 7.1 创建销售单

```
POST /api/v1/sales
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| items | object[] | 是 | 商品列表（IMEI级） |
| items[].imei | string | 是 | IMEI号码 |
| items[].unit_price | number | 是 | 销售单价 |

| customer_phone | string | 否 | 客户电话 |
| remark | string | 否 | 备注 |

5. 写入库存流水（change_type='sale_out'）
