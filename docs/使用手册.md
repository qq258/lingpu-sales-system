## 一、系统入门

### 1.1 系统简介

本系统包含两个客户端和一个后端服务：

| 客户端 | 访问地址 | 适用人群 | 主要功能 |
|--------|---------|---------|---------|
| **管理后台** | `http://服务器IP:5173` | 老板、店长、管理员 | 全部功能管理（品牌型号、供应商、调货、盘点、用户管理等） |
| **门户网站** | `http://服务器IP:5174` | 店员、收银员、仓管员 | 日常快捷操作（看板、入库、查库存、开单、销售记录） |

- 两个客户端共享同一套后端数据，操作实时同步
- 各门店数据隔离，店员只能看到本店数据
- 老板（超级管理员）可查看全部门店数据

### 1.2 如何登录

**管理后台登录：**
1. 在浏览器地址栏输入 `http://服务器IP:5173`
2. 输入用户名和密码
3. 点击**登录**按钮
4. 系统自动识别角色和所属门店，进入数据看板

**门户网站登录：**
1. 在浏览器地址栏输入 `http://服务器IP:5174`
2. 输入用户名和密码
3. 点击**登录**按钮
4. 进入数据看板首页

> 提示：如果记不住地址，可以问店长索要准确的访问地址。建议将网址添加到浏览器收藏夹。

### 1.3 账号与角色说明

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| **超级管理员**（老板） | 系统最高权限 | 全部功能；可查看和操作所有门店数据；可管理用户和门店 |
| **店长**（store_admin） | 各分店负责人 | 本店全部功能；可管理本店用户；调货申请/接收 |
| **店员**（operator） | 日常业务操作员 | 本店产品查询、进货、库存查询、销售开单、打印小票、看板查看 |

> 每个账号绑定所属门店。超级管理员不绑定门店，可切换查看任意门店。

### 1.4 门店切换

超级管理员登录后，可以在页面顶部的门店切换器中切换不同门店，查看该门店的数据。

- 门店切换仅影响数据查看范围
- 切换后，看板、库存、销售记录等页面将显示对应门店的数据

### 1.5 系统要求

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome、Edge 最新版本 |
| 网络 | 需通过 Tailscale 组网访问服务器 |
| 打印（可选） | 需要打印小票的电脑需安装 CLodop 打印插件 |
| 扫码枪（可选） | USB 扫码枪即插即用，自动输入 IMEI |

## 二、基础数据管理

> 本章操作均在**管理后台（端口 5173）**进行，需要店长或超级管理员权限。

### 2.1 品牌管理

**新增品牌：**
1. 进入管理后台 > **品牌型号管理**
2. 在品牌区域点击**新增品牌**
3. 输入品牌名称（如"华为"、"苹果"）和描述（可选）
4. 点击**确认**保存

**编辑品牌：**
1. 在品牌列表中点击目标品牌的**编辑**按钮
2. 修改名称或描述
3. 点击**确认**保存

**删除品牌：**
1. 在品牌列表中点击目标品牌的**删除**按钮
2. 系统会检查该品牌下是否有型号
3. 确认无型号后即可删除

> 注意：删除品牌前必须先删除该品牌下的所有型号。

### 2.2 型号管理

型号是系统中最核心的基础数据，包含手机的具体配置信息。

**新增型号：**
1. 进入管理后台 > **品牌型号管理**
2. 在型号区域点击**新增型号**
3. 填写以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 所属品牌 | 是 | 从下拉列表中选择 |
| 型号名称 | 是 | 如"iPhone 17 Pro Max" |
| 颜色 | 否 | 如"深空黑"、"星光色" |
| 运行内存（RAM） | 否 | 如"8GB"、"12GB" |
| 存储容量（ROM） | 否 | 如"256GB"、"512GB" |
| 销售价格 | 是 | 销售给客户的价格 |
| 进货成本价 | 否 | 进货时的成本价 |
| 厂家条码 | 否 | 包装盒上的 EAN-13 条码，必须唯一 |
| 上市年份 | 否 | 如 2026 |
| 操作系统 | 否 | iOS / Android |
| 网络制式 | 否 | 5G / 4G |
| 屏幕尺寸 | 否 | 如 6.7 英寸 |
| CPU | 否 | 处理器型号 |
| 电池容量 | 否 | 如 4500mAh |
| 描述 | 否 | 补充说明 |

4. 填写完毕后点击**确认**保存

**编辑/删除型号**操作方式与品牌管理类似。

> 提示：
> - 厂家条码录入后不可修改（系统唯一约束），录入时请仔细核对
> - 型号是全门店共享的基础数据，任一门店新增的型号所有门店都能看到

### 2.3 供应商管理

**新增供应商：**
1. 进入管理后台 > **供应商管理**
2. 点击**新增供应商**
3. 填写供应商名称、联系人、联系电话、地址和备注
4. 点击**确认**保存

**编辑/删除供应商**操作方式类似。

## 三、采购入库

### 3.1 入库前准备

在入库前，请确保系统中已存在以下数据：
- ✅ 相关**品牌和型号**已录入（如未录入，找店长操作）
- ✅ **供应商**信息已录入
- ✅ 仓库有足够的 IMEI 编码（扫码枪或记录好的 IMEI 列表）

### 3.2 门户网站快捷入库（推荐）

适合日常快速入库操作，一步到位。

1. 登录门户网站，点击顶部导航的**入库**，或从看板点击**入库**快捷按钮
2. **选择供应商**：从下拉列表中选择本次进货的供应商
3. **选择商品**：依次选择品牌和型号，系统会自动显示该型号的预设价格
4. **录入 IMEI**（两种方式）：
   - **逐台录入**（推荐配合扫码枪）：
     - 光标聚焦 IMEI 输入框
     - 用扫码枪扫描手机包装盒上的 IMEI 条码
     - 系统自动填入 IMEI，按 `Enter` 键添加到清单
     - 可选填写 IMEI2（双卡）和 SN 码（序列号）
   - **批量粘贴**：
     - 切换到"批量粘贴"模式
     - 复制多行 IMEI 数据粘贴到文本框中
     - 格式：每行一条，IMEI2 和 SN 可选
     ```
     861234567890101,861234567890102,SN123456
     861234567890103,,SN123457
     861234567890104
     ```
     - 点击**解析并添加**
5. 在入库清单中核对已添加的手机（型号、IMEI、价格）
6. 确认无误后，点击**确认入库**
7. 系统提示"入库成功"，库存即时更新

> 提示：使用扫码枪时，确保光标在 IMEI 输入框内，扫码后自动填充并回车，无需手动点击鼠标。

### 3.3 管理后台完整入库

适合需要管理入库单状态的场景，支持先保存后确认。

1. 进入管理后台 > **入库管理**
2. 点击**新建入库单**
3. 选择供应商，填写备注
4. 添加商品：选择品牌 → 型号 → 输入单价 → 录入 IMEI
5. 保存入库单（状态为"待确认"）
6. 在入库单列表中，找到该入库单，点击**确认入库**
7. 系统校验 IMEI 唯一性后完成入库

### 3.4 IMEI 录入方式说明

| 方式 | 适用场景 | 操作要点 |
|------|---------|---------|
| 扫码枪逐台录入 | 有实物手机，逐台扫码 | 光标在输入框，扫码后自动回车添加 |
| 手动逐台录入 | IMEI 记录在纸上或表格中 | 输入 IMEI 后按回车 |
| 批量粘贴 | 有电子表格，一次录入多台 | 按格式粘贴，系统自动解析 |

### 3.5 IMEI2 和 SN 码录入说明

- **IMEI2**：双卡双待手机有第二个 IMEI 号，录入后方便售后查询
- **SN 码**：手机序列号，厂家售后保修时需要
- 这两个字段为可选项，入库时未录入，在销售开单时也可以补录

## 四、库存管理

### 4.1 库存查询

**门户网站查库存：**
1. 登录门户网站，点击**查库存**
2. 在搜索框输入 IMEI 号码（可用扫码枪扫描）
3. 系统实时显示该手机的库存信息：型号、颜色、存储、IMEI 状态、所属门店
4. 也可以按品牌、型号筛选查看

**管理后台查库存：**
1. 进入管理后台 > **库存查询**
2. 支持按关键词（型号名称、条码、颜色）搜索
3. 支持按品牌、型号筛选
4. 以表格形式展示各型号的实时库存数量

### 4.2 IMEI 库存查询

1. 进入管理后台 > **库存查询** > **IMEI 库存**
2. 可搜索条件：IMEI 号、型号名称、厂家条码
3. 查看结果包含：
   - IMEI、IMEI2、SN 码
   - 当前状态（`in_stock` 在库 / `sold` 已售）
   - 所属门店
   - 入库时间

### 4.3 期初库存录入

> 适用于新门店首次启用系统时初始化库存，需要店长或超级管理员权限。

1. 进入管理后台 > **期初库存**
2. 选择型号，填写数量
3. 确认后系统写入库存，标记来源为"期初"
4. 支持批量录入多个型号

### 4.4 库存盘点

用于核对实际库存与系统库存是否一致。

**创建盘点单：**
1. 进入管理后台 > **库存盘点**
2. 点击**新建盘点单**
3. 录入各型号的账面数量（系统自动带出）和实际盘点数量
4. 系统自动计算差异
5. 保存盘点单（状态为"待审核"）

**审核盘点单：**
1. 在盘点单列表中，找到待审核的盘点单
2. 点击**审核**
3. 系统根据差异自动调整库存（盘盈补入、盘亏扣减）
4. 写入库存流水记录

> 注意：
> - 仅可删除"待审核"状态的盘点单
> - 审核后不可撤销，盘点前请仔细核对实盘数据

### 4.5 库存流水

查看每次库存变动的详细记录。

1. 进入管理后台 > **库存流水**
2. 支持按型号、变动类型筛选
3. 每条记录包含：变动类型、变动前后数量、操作人、操作时间

**变动类型说明：**

| 类型 | 含义 |
|------|------|
| initial | 期初录入 |
| purchase_in | 采购入库 |
| sale_out | 销售出库 |
| transfer_out | 调货出库 |
| transfer_in | 调货入库 |
| transfer_cancel | 调货取消回滚 |
| check | 盘点调整 |

### 4.6 低库存预警

- 系统默认低库存阈值为 **10 台**
- 在看板和库存页面会显示低库存商品列表
- 库存数量低于 10 台的型号会高亮提示

## 五、销售开单

### 5.1 门户网站销售开单

适合日常快速开单，操作流程简洁。

1. 登录门户网站，点击顶部导航的**开单**，或从看板点击**开单**快捷按钮
2. **扫码添加商品**：
   - 光标聚焦 IMEI 输入框
   - 用扫码枪扫描手机包装上的 IMEI 条码
   - 系统自动校验 IMEI 是否在本店且状态为在库
   - 校验通过后自动加入购物车
   - 也可以手动输入 IMEI 后按回车
3. **重复步骤 2**，添加同单中的其他商品
4. **核对购物车**：检查商品名称、IMEI、价格是否正确
   - 价格可手动调整
   - 可删除购物车中的商品
5. **填写结算信息**：
   - 实收金额（默认等于商品总价，可手动修改）
   - 客户姓名、电话、地址（可选）
6. 点击**确认开单**
7. 系统自动生成订单编号，IMEI 标记为已售，库存扣减
8. 保存成功后，可选择**打印小票**

### 5.2 管理后台销售开单

功能更完整，适合需要更多操作选项的场景。

1. 进入管理后台 > **销售开单**
2. 操作流程与门户网站一致，额外支持：
   - IMEI2 / SN 码补录（在购物车中直接编辑）
   - 更详细的商品信息展示

### 5.3 IMEI 实时校验说明

扫码或输入 IMEI 后，系统会自动校验：

| 校验项 | 不通过的提示 |
|--------|-------------|
| IMEI 是否存在 | "未找到该 IMEI 对应的库存" |
| IMEI 是否在本门店 | "该 IMEI 不属于本店" |
| IMEI 状态是否为在库 | "该商品已售出" |
| IMEI 是否已在购物车中 | "该 IMEI 已在购物车中" |

校验通过后商品自动加入购物车。

### 5.4 IMEI2 / SN 码补录

如果入库时未录入 IMEI2 或 SN 码，在销售开单时可以补录：
1. 在购物车中找到该商品
2. 点击 IMEI2 或 SN 码旁的**待补录**标签
3. 输入对应编号
4. 点击确认

### 5.5 打印小票

**首次使用需要安装 CLodop 插件：**
1. 在需要打印的电脑上访问 CLodop 官网下载并安装
2. 安装后重启浏览器
3. 系统会自动检测打印插件

**打印操作：**
1. 开单成功后，系统弹出打印预览
2. 点击**打印**按钮
3. 选择打印机
4. 确认打印

**补打小票：**
1. 进入**销售记录**
2. 找到需要补打的订单
3. 点击**打印**按钮
4. 按上述步骤操作

> 小票内容包含：门店名称、商品明细、IMEI、金额、销售时间等。

### 5.6 销售记录查询

**门户网站查看记录：**
1. 登录门户网站，点击顶部导航的**记录**
2. 按日期范围筛选（默认显示今日记录）
3. 每条记录以卡片形式展示：订单号、商品信息、金额、时间
4. 点击可查看详情（含 IMEI 明细）

**管理后台查看记录：**
1. 进入管理后台 > **销售记录**
2. 支持按日期范围、门店（超级管理员）筛选
3. 表格形式展示，支持查看详情和补打小票

## 六、门店调货

> 本章操作均在**管理后台（端口 5173）**进行，需要店长或超级管理员权限。

### 6.1 调货流程概述

调货是将一个门店的库存转移到另一个门店，流程如下：

```
发起调货申请 → 来源门店出库 → 目标门店入库 → 完成
  （pending）    （outbound）    （completed）
```

### 6.2 发起调货申请

1. 进入管理后台 > **调货管理**
2. 点击**发起调货**
3. 填写调货信息：
   - **目标门店**：选择要调往的门店
   - **商品型号**：选择要调货的型号
   - **数量**：需要调出的数量
   - **备注**：可选说明
4. 系统自动校验来源门店库存是否充足
5. 确认后提交，调货单状态为"待调出"

### 6.3 调货出库（来源门店操作）

1. 进入管理后台 > **调货管理**
2. 在调货单列表中，找到状态为"待调出"的调货单
3. 点击**确认出库**
4. 系统扣减来源门店库存
5. 调货单状态变为"已调出"
6. 写入库存流水（change_type = transfer_out）

### 6.4 调货入库（目标门店操作）

1. 进入管理后台 > **调货管理**
2. 在调货单列表中，找到状态为"已调出"的调货单
3. 点击**确认收货**
4. 系统增加目标门店库存
5. 调货单状态变为"已完成"
6. 写入库存流水（change_type = transfer_in）

### 6.5 取消调货

| 调货单状态 | 是否可以取消 | 取消后处理 |
|-----------|-------------|-----------|
| 待调出 | 是 | 直接取消，无库存影响 |
| 已调出 | 是 | 取消后自动回滚库存（扣减目标门店，恢复来源门店） |
| 已完成 | 否 | 不可取消 |

### 6.6 调货记录查询

1. 进入管理后台 > **调货管理**
2. 可按状态（全部/待调出/已调出/已完成）筛选
3. 可按门店（来源店或目标店）过滤
4. 查看调货详情

## 七、数据统计与看板

### 7.1 门户网站数据看板

登录门户网站后，首页即为数据看板，包含以下区域：

**统计卡片（顶部）：**
| 卡片 | 说明 |
|------|------|
| 今日销售额 | 当天所有已售订单的总金额 |
| 今日单数 | 当天完成的销售订单数量 |
| 低库存数量 | 库存低于 10 台的型号数量 |

## 九、常见问题与故障排除

### 9.1 登录问题

**Q：忘记密码怎么办？**
- 联系店长或超级管理员，在**用户管理**中重置密码

**Q：登录提示"账号已禁用"**
- 联系店长确认是否账号被禁用
- 如刚离职员工可能会被禁用账号

**Q：登录后看不到任何数据**
- 确认账号已绑定门店（超级管理员需手动切换门店）
- 联系系统管理员检查账号权限

### 9.2 IMEI 相关问题

**Q：扫码枪扫码后无反应**
- 检查光标是否在 IMEI 输入框中
- 检查扫码枪是否连接正常（打开记事本测试扫码）
- 检查 IMEI 条码是否清晰完整

**Q：提示"IMEI 已存在"**
- 该 IMEI 已经在系统中存在，可能是重复扫码
- 核对实物手机上的 IMEI 编号

**Q：提示"IMEI 已售出"**
- 该手机已经在销售记录中
- 检查是否误操作或扫码了已售手机的包装盒

**Q：扫码后提示"未找到该 IMEI 对应的库存"**
- 该 IMEI 不在本门店库存中
- 可能是其他门店的库存，或该 IMEI 从未入库

### 9.3 库存不一致

**Q：实际库存和系统库存对不上**
1. 进入**库存流水**查看最近变动记录
2. 核对是否有误操作（如重复入库、出库未成功等）
3. 进行**库存盘点**，以实际盘点数为准审核调整

**Q：调货后库存未更新**
1. 检查调货单状态是否已完成
2. 如果状态为"已调出"但目标门店未收到，联系目标门店操作"确认收货"
3. 如果状态为"待调出"，进行"确认出库"操作

### 9.4 打印问题

**Q：点击打印没反应**
- 确认电脑上已安装 CLodop 插件
- 确认 CLodop 服务正在运行（桌面右下角有 CLodop 图标）
- 重启浏览器后重试

**Q：打印内容排版错乱**
- 确认使用的是 CLodop 支持的打印机
- 联系技术人员调整打印模板

**Q：打印出来的小票缺字**
- 检查打印机是否缺纸或碳带
- 联系技术人员调整打印参数

### 9.5 调货异常

**Q：发起调货提示"库存不足"**
- 确认来源门店的该型号库存数量是否足够
- 检查是否有其他未完成的调货单占用了库存

**Q：调货后想取消**
- 状态为"待调出"：直接取消，无影响
- 状态为"已调出"：取消后库存自动回滚
- 状态为"已完成"：无法取消，需要重新调回

### 9.6 数据工具使用场景

以下情况可联系超级管理员使用数据工具处理：
- 因误操作导致的大量重复 IMEI 数据
- 需要批量修正库存数据
- 系统测试数据清理

> 数据工具操作不可逆，操作前请务必备份数据库。
