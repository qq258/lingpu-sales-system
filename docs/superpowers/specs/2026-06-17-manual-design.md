# 手机销售管理系统 — 使用手册设计文档

## 1. 概述

### 1.1 背景
当前门户网站（portal）的 Manual.vue 页面仅展示"建设中"占位内容，管理后台（client）也无使用手册入口。系统已有完整的功能模块，但一线店员和管理员缺乏操作指导文档。

### 1.2 目标
生成一份覆盖全系统的使用手册，包含：
- **网页版**：替换 portal 中 Manual.vue 的占位内容，可在浏览器中直接浏览
- **文档版**：Markdown 格式，方便打印和导出

### 1.3 读者
- **门店店员**（operator）：关注日常操作（入库、查库存、开单、看板）
- **店长/管理员**（store_admin / super_admin）：关注管理功能（调货、盘点、用户管理、数据管理）
- 手册结构兼顾两者，按业务流程组织，每个模块下区分"门店操作"和"管理操作"

## 3. 技术方案

### 3.1 网页版（Manual.vue）

- 沿用现有的 `<div class="page-container">` 布局框架
- 使用 Element Plus 的 `el-collapse` 折叠面板组件实现章节展开/收起
- 每个章节为一个折叠面板，点击标题展开内容
- 章节内部使用 Markdown 渲染（通过 `marked` 库或手动解析）
- 保持门户网站的大字号、磨砂玻璃设计风格
- 页面路由：已在 router 中配置 `/manual` 路由

### 3.2 文档版（Markdown）

- 保存为 `docs/使用手册.md`
- 使用标准的 GFM 格式
- 含表格、代码块、列表等元素
- 可直接用 Markdown 编辑器打开或打印

### 3.3 内容来源

手册内容基于以下资料编写：
- 需求规格说明书.md
- 概要设计文档.md
- 详细设计说明书.md
- 门户客户端设计文档.md
- 源代码（API 路由、前端视图组件）

## 4. 风格规范

### 4.1 语言风格
- 简洁明了，避免技术术语
- 操作步骤用数字序号（1. 2. 3.）
- 每个操作步骤以动词开头（点击、选择、输入、确认）

### 4.2 格式约定
| 元素 | 格式 | 示例 |
|------|------|------|
| 页面/按钮名称 | **加粗** | 点击**确认入库** |
| 路径导航 | > 分隔 | 管理后台 > 库存管理 > 库存盘点 |
| 提示/警告 | 引用块 | > 注意：IMEI 一旦确认不可修改 |
| 键盘操作 | `行内代码` | 按 `Enter` 键 |
| 代码/示例 | 代码块 | 批量粘贴格式示例 |
| 术语首现 | *斜体* | *IMEI* 是设备的唯一标识 |

### 4.3 注意事项
- 不包含具体价格数据，使用"预设价格"等通用描述
- 不包含敏感信息（密码、密钥等）
- 每个操作步骤描述不超过 5 步
- 复杂流程拆分为多个小节
