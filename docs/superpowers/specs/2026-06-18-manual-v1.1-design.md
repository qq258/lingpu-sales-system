## 1. 概述

## 3. 各变更详情

### 3.1 第三章 — 追加 3.6 无库存销售-入库场景

**位置**:在 3.5 IMEI2 和 SN 码录入说明之后

**标题**:`3.6 无库存销售-入库场景(门户网站)`

**适用**:历史商品在售出时尚未登记入库,需要现录现卖。

**操作步骤**:
1. 门户网站 > **开单** 页面,顶部"销售类型"切换至 **无库存销售**
2. **搜索品牌型号**:输入框支持模糊搜索(匹配品牌名 + 型号名)
3. 选中已有型号(自动带入预设售价)或下拉选"未找到匹配项,将保存为新品牌型号"
4. **录入 IMEI**:三个输入框依次录入 IMEI1 / IMEI2 / SN(支持扫码枪逐字段自动跳焦)
5. 输入**售出价格** → 点**添加至购物车**
6. 收款完成后,系统自动调后端接口新建品牌型号 + 库存记录

**注意**:> 注意:无库存销售产生的库存记录会立即出现在库存查询中,后续可正常补录 IMEI2/SN。

### 3.2 第四章 — 追加 4.7 库存查询支持 IMEI2 / SN 码搜索与补录

**位置**:在 4.6 低库存预警之后

**标题**:`4.7 库存查询支持 IMEI2 / SN 码搜索与补录`

**4.7.1 搜索范围扩展**
- 搜索框支持输入:IMEI / IMEI2 / SN 码 / 商品名称 / 厂家条码
- 搜索框 placeholder 改为:`搜索 IMEI / IMEI2 / SN码 / 商品名称...`

**4.7.2 卡片展开与补录**
- 库存卡片默认展示品牌、型号、规格、IMEI
- 点击卡片上的**展开按钮**,显示 IMEI2 和 SN 码
- 若某字段为空,显示**补录**按钮
- 点补录 → 行内变为输入框 → 输入后回车 → 调用后端 `PUT /inventory/imei/:id` 保存
- 保存后卡片自动刷新,显示新值

**4.7.3 补录数据流**
```
用户点击补录 → 内联输入 → PUT /inventory/imei/:id { imei2?: string, sn_code?: string }
  → 后端更新 wh_inventory_imei 表
  → 返回更新后记录 → 卡片刷新
```

### 3.3 第五章 — 追加 5.7 无库存销售(开单场景)

**位置**:在 5.6 销售记录查询之后

**标题**:`5.7 无库存销售(开单场景)`

> 引用注:与 3.6 互补:3.6 描述入库逻辑,5.7 描述开单完整流程。

**5.7.1 切换销售类型**
- 在门户网站开单页顶部,新增 radio 切换:`普通销售` / `无库存销售`
- 默认 `普通销售`;切换时若购物车有商品,弹窗确认后清空

**5.7.2 无库存销售模式**
- **品牌型号搜索**:支持模糊搜索,无匹配时允许用户输入并保存为新品牌型号(弹 `SaveBrandDialog` 二次确认)
- **保存品牌型号**:弹窗中按第一个空格自动分割为品牌 + 型号,用户可调整;只允许点"确认"(无取消)
- **IMEI/SN 录入**:扫码枪逐字段自动跳焦
- **购物车**:每个商品展示 IMEI1/IMEI2/SN,未录入字段显示`[待录入]`标签,点击行内编辑
- **结算**:与普通销售一致(合计、实收、找零、客户信息、备注)

**5.7.3 提交后自动建库**
- 收款确认后,后端 `POST /sales/no-stock` 一次性完成:新建品牌型号 + 新建库存记录 + 创建销售单 + 扣减库存

### 3.7 新增 第十章 售后记录

**位置**:在第九章之后,作为新的最大章(章末位置)

**章首注**:本章操作在**管理后台(端口 5173)「售后管理」** 完成新建与处理;**门户网站(端口 5174)** 通过顶部「售后」Tab 查看工单与处理进度。

**10.1 售后场景概述**

三种售后类型对照表:

| 场景 | 说明 | 库存影响 |
|------|------|---------|
| 维修(repair) | 当场修 / 店里修 / 返厂修 | 维修期间手机状态置为 `维修中`,完结后恢复 `已售` |
| 换货(exchange) | 从库存换出另一台(可不限制同品牌型号) | 原手机 → `已换货`;新手机 `在库` → `已售` |
| 退款(refund) | 原手机退回,退款给客户 | 原手机 → `已退货` |

**10.2 售后工单状态流转**

两个独立维度:

- **主状态**:`待处理` → `检测中` → (`维修中` → `已维修` / `换货中` → `已换货` / `退款中` → `已退款`) → `已完成`,可任意阶段 `已取消`
- **供应商子状态**:`无需供应商` → `待联系供应商` → `供应商处理中` → `供应商已处理`

> 提示:供应商子状态独立运行,可在主状态的任意阶段触发,完成后回到主流程继续。

**10.3 新建工单**

操作步骤(管理后台):
1. 进入管理后台 > **售后管理** > **新建工单**
2. **扫码/搜索 IMEI**:光标聚焦 IMEI 输入框,扫码或输入后回车,系统自动匹配手机信息(品牌/型号/颜色/存储/原销售记录)
3. 填写**客户信息**:姓名、电话、地址(选填)
4. 填写**故障描述**(必填)
5. 点击**创建工单**,系统自动生成工单号(格式:`ASyyMMddHHmmss` + 3 位随机)

> 提示:未找到该 IMEI 记录则无法创建工单(该手机未在本店售出)。

**10.4 维修处理**
1. 进入工单详情,选择处理方式 → **维修**
2. 选择**维修级别**:`当场` / `店里` / `返厂`
3. 填写费用(可选)和费用说明
4. 提交后手机 IMEI 状态置为 `维修中`
5. 维修完成后,在操作区点**已维修**,手机状态恢复 `已售`

**10.5 换货处理**
1. 进入工单详情,选择处理方式 → **换货**
2. 填写**换货新机**:选择新手机型号 + 录入新 IMEI
3. 提交后原手机状态置为 `已换货`,新手机从库存扣减
4. 完成后点**已换货**,工单进入 `已完成`

> 注意:换货时新手机不限品牌型号,但必须在本店库存中。

**10.6 退款处理**
1. 进入工单详情,选择处理方式 → **退款**
2. 填写退款金额和说明
3. 提交后原手机状态置为 `已退货`
4. 完成后点**已退款**,工单进入 `已完成`
5. 原销售单状态保持不变(已售事实不变)

**10.7 供应商沟通**

适用于返厂维修/退货给供应商等场景:
1. 在工单详情操作区,设置**供应商状态**为`待联系`
2. 在底部日志输入框填写沟通内容,点击**添加日志**(action = `supplier_contact`)
3. 供应商处理中时切换状态为`供应商处理中`
4. 供应商处理完毕,切换为`供应商已处理`,并填写**供应商处理结果**

**10.8 工单查询与详情**

**查询(管理后台):** 售后管理列表,支持按工单号/IMEI/客户姓名/电话/状态/处理方式/日期范围筛选,支持分页。超级管理员可按门店筛选。

**详情(管理后台 / 门户网站通用):**
- 顶部三卡片:手机信息 | 客户信息 | 当前状态(带状态标签)
- 故障描述
- 处理操作区(检测结果/处理方式/费用/供应商信息)
- 操作日志流(时间线展示)

**10.9 角色与权限**

| 角色 | 可执行操作 |
|------|----------|
| 超级管理员 | 全部门店工单查看 + 处理 + 统计 |
| 店长 | 本店工单查看 + 处理 + 新建 |
| 店员 | 本店工单查看 |

> 提示:售后工单号一旦生成不可修改,删除仅限 `待处理` 状态。

**章末注**:
> 备注:售后记录是 v1.1 新增模块,操作涉及库存状态变更,请处理前先确认工单对应的手机实物状态。
